# Transking v4.3 使用指南

## 1. 概述

Transking 是一款专为长篇文本（如小说、专业文档）设计的批量 LLM 翻译工具。它通过"分块翻译 + 进度落盘 + 看门狗监控"的组合拳，解决了大规模翻译任务中的稳定性难题。

核心优势：

* 零会话污染：直接通过 19000 端口调用 LLM 代理，不在主会话产生冗余记录。

* 高鲁棒性：天然应对 API 速率限制（403/429）与系统级硬性超时（SIGKILL）。

* 智能续传：看门狗重启时自动推断起始编号，无需手动指定 `--start`。

***

## 2. 快速开始

### 单文件翻译

```bash
python auto_translate.py --file "C:\Users\evezero\transking\source\sample.txt" --output-dir "C:\Users\evezero\transking\output"
```

### 批量翻译（指定编号范围）

```bash
python auto_translate.py \
  --source-dir "C:\Users\evezero\transking\source" \
  --output-dir "C:\Users\evezero\transking\output" \
  --start 1 \
  --end 50 \
  --model pool-deepseek-v4-pro
```

## 3. 速率限制与 2 小时硬性超时应对

### 3.1 速率限制 (Rate Limit)

当 API 返回 403 或 429 错误时，脚本会创建 `_rate_limit_flag` 标记并主动退出。

* `_rate_limit_flag` 仅为信息标记，供用户查看限流发生时间，不影响看门狗决策。

* 看门狗检测到进程退出+项目未完成时会直接重启，若仍限流则翻译进程再次退出，看门狗下一轮再试。

### 3.2 2 小时硬性超时 (SIGKILL)

腾讯 QClaw 系统的 exec 命令有 2 小时运行上限，超时后进程会被强行终止。

* 应对方案：

  1. 断点续传：脚本每完成一个分块都会即时落盘，重启后会自动跳过已完成的内容。

  2. 看门狗全自动重启：建议配置 Cron 任务定时运行看门狗。

     **看门狗 v5.3 双重检测：**

     ```
     项目完成标志？→ 结束
       ↓
     有 .part 文件？
       ├─ 是 → 卡住(>300s)？→ 重启
       └─ 否 → 文件数对不上？→ 重启
       ↓
     一切正常，无需干预
     ```

     **智能续传**：重启时不再固定 `--start 1`，而是自动推断：

     * 有 `.part` 文件 → 从最新的 `.part` 文件对应编号开始

     * 无 `.part` 文件 → 从已完成的最大编号 +1 开始

  ⚠️ **delivery 必须设为 `"none"`**：在 openclaw 控制台创建 cron job 时，delivery 选项不要用 `announce`——若设为 announce 且未指定 target，连续失败会触发系统隐性退避（实际间隔从 10 分钟变成 1 小时）。务必设为 `none`，看门狗才能稳定按设定频率运行。

  查看 cron 列表: `openclaw cron list`
  查看运行记录: `openclaw cron runs <jobId>`

## 4. 完成后自动质检 (Post-Check)

项目完成后，脚本会自动执行质检逻辑：

1. **`<think>` 标签清理**：仅检测并移除 `<think>...</think>` 标签及其包裹的 LLM 思考内容。

2. **不再删除英文分析性语句**：因审查等原因可能保留英文原文，粗暴删除英文会导致内容丢失。宁可保留少量异常信息，也不误删合法内容。

3. 编码容错：质检读取文件时，若遇到非法 UTF-8 字节，会自动降级为容错读取（`errors="replace"`），不会中断质检流程。

## 5. 常见参数调优

| 参数            | 建议值     | 说明                                       |
| :------------ | :------ | :--------------------------------------- |
| --chunk-max   | 1500    | 单块字符数。API 报错多时建议调大（减少请求总数）。              |
| --chunk-delay | 15 - 60 | 块间延迟（含文件最后一个块，自然覆盖文件间过渡）。频繁触发速率限制时请增加此值。 |

~~`--file-delay`~~：已移除。文件间无额外延迟，跳过已完成文件时零等待。
