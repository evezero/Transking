# Transking v4.2 / 看门狗 v5.3

## 核心能力 (Overview)

提供经过验证的批量 LLM 翻译标准化工作流，解决长篇文本（如小说、专业文档）翻译中的进程中断、API 速率限制等痛点。

## 核心机制 (Core Mechanisms)

### 1. 速率限制与断点续传 (Rate Limit & Resumption)

* 四级应对策略：

  1. 块重试：单块翻译失败会自动重试 3 次，并采用指数退避（60s → 180s）。

  2. 主动退出：遇到 403/429（速率限制）时，脚本立即创建标记文件并退出，避免无效重试导致封号。

  3. 看门狗监控（v5.3）：`_translate_watchdog.py` 每 20 分钟被 cron 触发一次，双重检测：

     **检测1 — .part 文件卡住**：检测 `.part` 文件最新修改时间，超过 300 秒未更新则判定为卡住，杀掉进程并重启翻译。

     **检测2 — 无 .part + 项目未完成**：当无 `.part` 文件时，对比源目录和输出目录的文件数。若文件数未对齐，直接重启翻译进程。

     **速率限制标记**：`_rate_limit_flag` 仅为信息标记（记录限流发生时间），供用户查看，不影响看门狗的重启决策。若重启后仍限流，翻译进程会再次退出，看门狗下一轮再试。

     **智能续传**：重启时不再固定 `--start 1`，而是自动推断起始编号：
     - 有 `.part` 文件 → 找输出目录中修改时间最新的 `.part` 文件，从该文件对应编号开始
     - 无 `.part` 文件 → 找输出目录中序号最大的已完成 `.txt` 文件，从其下一个序号开始

  4. 速率限制容忍：重启后若仍遇 403/429，看门狗会在下一轮（20 分钟后）再次检测并重试，最坏情况是每 20 分钟尝试一次。

* 块级断点续传：翻译过程中实时保存 `.part` 内容和进度标记。任务被系统（如 2 小时硬性超时）强行终止后，重新运行同一命令将精准从断点处继续，不浪费任何 Token。

### 2. 完成后自动质检修复 (Post-Completion Repair)

项目完成后，脚本会自动执行二次扫描与修正：

* LLM 思考过程清理：自动检测并删除译文中误混入的分析性文字（如 "The user wants me to..."）。

## 执行工作流 (Execution Workflow)

### 1. 启动翻译

```bash
python auto_translate.py --source-dir <源目录> --output-dir <目标目录> --chunk-max 1500 --chunk-delay 30
```

参数说明：
- `--chunk-max`：默认 1500
- `--chunk-delay`：默认 30s，API 严格时调大。每个块（含文件最后一个块）翻译后均冷却，自然覆盖文件间过渡
- ~~`--file-delay`~~：已移除。文件间无额外延迟，跳过已完成文件时零等待

### 2. 启动看门狗 Cron【必须】

翻译任务启动后，**必须**立即创建看门狗 cron 任务。QClaw exec 有 7200 秒运行上限，看门狗负责自动重启中断的翻译进程。

**cron 配置要求：**

```json
{
  "name": "Transking Watchdog - <项目名>",
  "schedule": { "kind": "every", "everyMs": 1200000 },
  "payload": {
    "kind": "agentTurn",
    "message": "直接执行以下命令，不要分析，不要解释：\npython <scripts目录>/_translate_watchdog.py --source-dir <源目录> --output-dir <目标目录>"
  },
  "delivery": { "mode": "none" }
}
```

**关键配置：**
- `delivery.mode` 必须为 `none`（设为 announce 会导致隐性退避）
- `message` 必须以 `"直接执行以下命令，不要分析，不要解释："` 开头
- 必须包含完整参数：`--source-dir` 和 `--output-dir`

**看门狗工作流程（v5.3）：**

```
项目完成标志？→ 结束
  ↓
有 .part 文件？
  ├─ 是 → 卡住(>300s)？→ 重启
  └─ 否 → 文件数对不上？→ 重启
  ↓
一切正常，无需干预
```
